# دليل رفع وتجهيز تطبيق "ارينا لايف" على استضافة Render 🚀

لقد قمنا بتجهيز وتهيئة المشروع بالكامل ليكون متوافقاً تماماً ومستعداً للرفع المباشر والتشغيل على منصة **Render** كخدمة ويب كاملة (Node.js Web Service) بأعلى كفاءة واستقرار.

---

## 🛠️ التحديثات والتحسينات التي تم إجراؤها للتوافق مع Render:

1. **دعم المنافذ الديناميكية (Dynamic PORTs):**
   تم تعديل خادم المشروع (`server.ts`) ليتعرف تلقائياً على المنفذ الذي تحدده خوادم Render الديناميكية عبر المتغير `process.env.PORT` بدلاً من فرض المنفذ الثابت `3000` مما يضمن نجاح اختبارات جهوزية الخدمة (Health Checks).
   
2. **بناء وتجميع الخادم آلياً (Production Bundling):**
   تم تحديث نصوص التشغيل في `package.json` لتجميع كود الخادم المكتوب بلغة TypeScript إلى ملف مضغوط وسريع التشغيل للغاية في مسار `dist/server.cjs` باستخدام أداة `esbuild`. يسرع هذا من تشغيل الخدمة على Render ويمنع أي أخطاء متعلقة باستيراد مكتبات Node.

3. **ملف الإعداد التلقائي (Render Blueprint):**
   قمنا بإنشاء ملف `render.yaml` في الجذر الرئيسي للمشروع. يتيح لك هذا الملف تهيئة الخدمة تلقائياً بضغطة زر واحدة دون الحاجة لإدخال الإعدادات يدوياً.

---

## 📋 خطوات رفع المشروع على Render (خطوة بخطوة):

### الطريقة الأولى: الاستيراد التلقائي باستخدام Blueprint (الأسهل والأسرع)
1. قم بإنشاء حساب مجاني أو تسجيل الدخول على [Render.com](https://render.com).
2. قم بربط حسابك في **GitHub** أو **GitLab** بمنصة Render.
3. ارفع كود المشروع الحالي إلى مستودع (Repository) خاص بك على GitHub.
4. في لوحة تحكم Render، اضغط على زر **New +** ثم اختر **Blueprint**.
5. اختر مستودع المشروع الخاص بك، وسيقوم Render تلقائياً بقراءة ملف `render.yaml` وضبط كافة الإعدادات كالتالي:
   - **نوع الخدمة:** Web Service
   - **أمر البناء (Build Command):** `npm install && npm run build`
   - **أمر البدء (Start Command):** `npm start`
6. اضغط على **Approve** لبدء الرفع والتشغيل فوراً!

---

### الطريقة الثانية: الإعداد اليدوي (Manual Setup)
إذا كنت تفضل تهيئة الإعدادات بنفسك يدوياً:
1. في لوحة تحكم Render، اضغط على زر **New +** ثم اختر **Web Service**.
2. اختر مستودع المشروع الخاص بك على GitHub.
3. قم بتعبئة البيانات كالتالي:
   - **Name:** `arena-live` (أو أي اسم تفضله)
   - **Language / Runtime:** `Node`
   - **Branch:** `main` (أو الفرع الافتراضي لكودك)
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm start`
4. اضغط على **Advanced** لإضافة المتغيرات البيئية (Environment Variables) التالية:
   - اضغط على **Add Environment Variable**:
     - **Key:** `NODE_ENV`
     - **Value:** `production`
   - (اختياري لتشغيل الذكاء الاصطناعي):
     - **Key:** `GEMINI_API_KEY`
     - **Value:** *[مفتاح الـ API الخاص بك من Google AI Studio]*
5. اضغط على **Create Web Service** في أسفل الصفحة لتبدأ المنصة ببناء مشروعك وتوفير رابط مباشر وآمن (HTTPS) لك.

---

## 💡 ملاحظات هامة للبث الحي على Render:
- **تجاوز قيود CORS والمحتوى المختلط:**
  واجهة التشغيل الذكية ستتعرف تلقائياً على أن المشروع يعمل بنظام Node.js كامل، وستقوم بتوجيه روابط البث غير المشفرة (HTTP) عبر الممر السريع والآمن المتكامل بداخل الخادم على المسار `/api/stream-proxy` لتفادي ظهور رسالة "القناة غير متاحة حالياً" وضمان عمل جميع القنوات بأعلى جودة وثبات.
